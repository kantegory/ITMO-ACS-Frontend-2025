# Лабораторная работа 2 — Работа с внешним API

## Описание
В этой лабораторной работе реализовано взаимодействие веб-приложения **RestaurantBooking** с внешним API (имитация через `JSON Server`). Приложение позволяет просматривать рестораны, бронировать столики и вести историю бронирований. Авторизация реализована через временное хранение текущего пользователя (LocalStorage) и проверку данных через API.

## Шаги выполнения

1. **Создание мокового API**
   - Установлен `JSON Server`.
   - Создан файл `db.json` с данными: пользователи, рестораны, бронирования.
   - API доступно по адресу `http://localhost:3000`.

2. **Подключение фронтенда к API**
   - На страницах `index.html`, `restaurant.html`, `account.html` и `history.html` реализован `fetch` для получения и отправки данных.
   - Пример: получение списка ресторанов и бронирований, отправка нового бронирования.

3. **Реализация авторизации**
   - Вход и регистрация пользователей через `login.html` и `register.html`.
   - Проверка введённых данных через API и сохранение текущего пользователя в LocalStorage на время сеанса.
   - Кнопка "Выйти" очищает текущую сессию.

4. **Просмотр ресторанов и бронирование**
   - Список ресторанов выводится с фильтрами по кухне, району и цене.
   - Возможность перехода на страницу конкретного ресторана и просмотра меню и отзывов.
   - Модальное окно для бронирования с отправкой данных на сервер.

5. **История бронирований**
   - На странице `account.html` и `history.html` выводятся только бронирования текущего пользователя.
   - Данные подгружаются с сервера через API.

6. **Подключение к API**
   - Все действия с пользователями и бронированиями теперь происходят через `JSON Server`.
   - LocalStorage используется только для хранения текущего сеанса (временное решение).

## Итог
- Приложение подключено к внешнему API.
- Реализована авторизация и управление сеансом пользователя.
- Работа с данными перенесена с локальных массивов на сервер.
- Можно фильтровать рестораны, просматривать меню, отзывы и историю бронирований.
